## 3. 部署方式一：VM 虚拟机

> 适用场景：在云服务器（阿里云/腾讯云/华为云）、VirtualBox/VMware 搭建的 Linux 虚拟机、或物理 Linux 主机上部署。

### 3.1 环境准备

#### 3.1.1 安装 Docker（CentOS / Rocky Linux）

```bash
# 卸载旧版本
sudo yum remove docker docker-client docker-client-latest docker-common \
                docker-latest docker-latest-logrotate docker-logrotate docker-engine

# 安装依赖
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 安装 Docker Engine + Compose 插件
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER
# 重新登录后生效
```

#### 3.1.2 安装 Docker（Ubuntu / Debian）

```bash
# 卸载旧版本
sudo apt remove docker docker-engine docker.io containerd runc

# 安装依赖
sudo apt update
sudo apt install -y ca-certificates curl gnupg

# 添加 Docker 官方 GPG 密钥
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 添加仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 将当前用户加入 docker 组
sudo usermod -aG docker $USER
# 重新登录后生效
```

#### 3.1.3 验证安装

```bash
docker --version          # 应显示 24.x+
docker compose version    # 应显示 2.20+
docker run hello-world    # 验证 Docker 可以正常运行
```

### 3.2 项目部署

#### 3.2.1 创建项目目录

```bash
# VM 上使用标准 Linux 路径
mkdir -p /opt/water_compus
cd /opt/water_compus
```

#### 3.2.2 上传/克隆项目文件

```bash
# 方式一：Git 克隆
git clone <仓库地址> .

# 方式二：scp 上传（从本机）
# 在本机执行: scp -r water_compus/* user@your-vm-ip:/opt/water_compus/

# 创建数据持久化目录
mkdir -p mysql/init mysql/data redis/data rabbitmq/data nacos/data minio/data
```

#### 3.2.3 启动全部容器

```bash
cd /opt/water_compus

# 启动（后台运行）
docker compose up -d

# 查看状态
docker compose ps

# 跟踪日志（确认无报错）
docker compose logs -f
```

> 首次启动会拉取镜像（约 3-5 分钟），MySQL 初始化完成后 Nacos 和 XXL-JOB 才会启动。

#### 3.2.4 导入业务数据库

```bash
# 导入 DDL（29张表）
docker exec -i huashui_mysql mysql -uroot -proot huashui_campus_life < huashui_DB/01_huashui_campus_life_ddl.sql

# 导入初始数据
docker exec -i huashui_mysql mysql -uroot -proot huashui_campus_life < huashui_DB/02_huashui_campus_life_init_data.sql
```

#### 3.2.5 配置防火墙（云服务器必须）

```bash
# 仅开放必要端口（按需调整）
sudo firewall-cmd --add-port=8848/tcp --permanent   # Nacos
sudo firewall-cmd --add-port=8081/tcp --permanent   # XXL-JOB
sudo firewall-cmd --add-port=3306/tcp --permanent   # MySQL（建议仅内网开放）
sudo firewall-cmd --reload

# 云服务器还需在控制台安全组中放行对应端口
```

## 4. 部署方式二：WSL

> 适用场景：在 Windows 10/11 上通过 WSL2 + Docker Desktop 部署，适合本地开发调试。

### 4.1 环境准备

#### 4.1.1 安装 WSL2

```powershell
# 在 PowerShell（管理员）中执行
wsl --install
# 默认安装 Ubuntu，重启电脑后完成
```

如果已安装 WSL1，升级到 WSL2：

```powershell
wsl --set-default-version 2
wsl --set-version Ubuntu 2
```

#### 4.1.2 安装 Docker Desktop

1. 从官网下载 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)
2. 安装后打开设置：**Settings → Resources → WSL Integration**
3. 开启 **Enable integration with my default WSL distro**
4. 勾选你的 Ubuntu 发行版
5. 点击 **Apply & Restart**

#### 4.1.3 验证安装

在 WSL 终端中执行（**不是 PowerShell**）：

```bash
docker --version          # 应显示 24.x+
docker compose version    # 应显示 2.20+
docker run hello-world    # 验证 Docker 可以正常运行
```

### 4.2 项目部署

> **重要**：将项目放在 WSL 内部文件系统中（如 `~/water_compus`），**不要**放在 `/mnt/c/`（Windows 文件系统），否则：
> - 容器挂载卷性能极差（跨文件系统 IO）
> - MySQL/Redis 等数据库可能出现文件锁错误
> - 部分镜像的权限检查会失败

#### 4.2.1 项目放对位置

```bash
# ✅ 正确：放在 WSL 内部
mkdir -p ~/water_compus
cd ~/water_compus

# ❌ 错误：不要放 /mnt/c/Users/... 下面
```

如果项目当前在 Windows 目录（如 `E:\Workspace\demos\water_compus`），复制进 WSL：

```bash
# 从 WSL 终端执行
cp -r /mnt/e/Workspace/demos/water_compus/* ~/water_compus/
```

#### 4.2.2 创建数据目录

```bash
cd ~/water_compus
mkdir -p mysql/init mysql/data redis/data rabbitmq/data nacos/data minio/data
```

#### 4.2.3 启动全部容器

```bash
cd ~/water_compus

# 启动（后台运行）
docker compose up -d

# 查看状态
docker compose ps

# 跟踪日志
docker compose logs -f
```

#### 4.2.4 导入业务数据库

```bash
# 导入 DDL
docker exec -i huashui_mysql mysql -uroot -proot huashui_campus_life < huashui_DB/01_huashui_campus_life_ddl.sql

# 导入初始数据
docker exec -i huashui_mysql mysql -uroot -proot huashui_campus_life < huashui_DB/02_huashui_campus_life_init_data.sql
```

#### 4.2.5 从 Windows 浏览器访问

WSL 内部启动的容器端口会自动映射到 Windows 的 `localhost`，直接用 Windows 浏览器访问即可：

| 服务 | Windows 浏览器访问 |
|------|--------------------|
| Nacos | `http://localhost:8848/nacos` |
| Sentinel | `http://localhost:8858` |
| RabbitMQ | `http://localhost:15672` |
| MinIO | `http://localhost:9001` |
| XXL-JOB | `http://localhost:8081/xxl-job-admin` |

#### 4.2.6 WSL 开机自启（可选）

Docker Desktop 开机自启后，在 WSL 中启动容器：

```bash
# 编辑 ~/.bashrc，添加别名
echo 'alias hsup="cd ~/water_compus && docker compose up -d"' >> ~/.bashrc
echo 'alias hsdown="cd ~/water_compus && docker compose down"' >> ~/.bashrc
echo 'alias hsps="cd ~/water_compus && docker compose ps"' >> ~/.bashrc
source ~/.bashrc

# 之后直接
hsup    # 启动全部服务
hsps    # 查看状态
```

## 7. 初始化业务数据库

容器启动后，`huashui_campus_life` 库是空的，需手动导入 DDL 和初始数据。

### 方式一：命令行导入（推荐）

```bash
# 导入 DDL（29张表结构）
docker exec -i huashui_mysql mysql -uroot -proot huashui_campus_life < huashui_DB/01_huashui_campus_life_ddl.sql

# 导入初始数据（校区/楼栋/配置/角色/权限）
docker exec -i huashui_mysql mysql -uroot -proot huashui_campus_life < huashui_DB/02_huashui_campus_life_init_data.sql
```

### 方式二：进入容器手动执行

```bash
# 复制 SQL 文件到容器
docker cp huashui_DB/01_huashui_campus_life_ddl.sql huashui_mysql:/tmp/
docker cp huashui_DB/02_huashui_campus_life_init_data.sql huashui_mysql:/tmp/

# 进入容器执行
docker exec -it huashui_mysql bash
mysql -uroot -proot huashui_campus_life < /tmp/01_huashui_campus_life_ddl.sql
mysql -uroot -proot huashui_campus_life < /tmp/02_huashui_campus_life_init_data.sql
exit
```

### 验证导入

```bash
docker exec -it huashui_mysql mysql -uroot -proot huashui_campus_life -e "SHOW TABLES;"
# 应显示 29 张表

docker exec -it huashui_mysql mysql -uroot -proot huashui_campus_life -e "SELECT id, campus_name, campus_code FROM sys_campus;"
# 应显示 3 个校区

docker exec -it huashui_mysql mysql -uroot -proot huashui_campus_life -e "SELECT COUNT(*) AS building_count FROM dorm_building;"
# 应显示 39
```

## 8. Spring Boot 项目配置

### 8.1 开发环境（本机 localhost）

Spring Boot 在本机（IDE）运行时，连接 Docker 容器的中间件：

```yaml
spring:
  application:
    name: huashui-gateway   # 按模块替换

  # Nacos 注册 & 配置中心
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
        namespace: public
        username: nacos
        password: nacos
      config:
        server-addr: localhost:8848
        file-extension: yaml

  # MySQL 数据源
  datasource:
    url: jdbc:mysql://localhost:3306/huashui_campus_life?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai
    username: root
    password: root
    driver-class-name: com.mysql.cj.jdbc.Driver

  # Redis
  data:
    redis:
      host: localhost
      port: 6379
      password: redis123

  # RabbitMQ
  rabbitmq:
    host: localhost
    port: 5672
    username: huashui
    password: huashui123

# Sentinel
spring.cloud.sentinel.transport.dashboard: localhost:8858

# MinIO
minio:
  endpoint: http://localhost:9000
  access-key: huashui
  secret-key: huashui123456

# XXL-JOB
xxl:
  job:
    admin:
      addresses: http://localhost:8081/xxl-job-admin
    accessToken: huashui_xxl_token
    executor:
      appname: huashui-executor
      port: 9999
```

### 8.2 Docker 内部通信（微服务也容器化时）

将 `localhost` 改为容器名（Docker Compose 网络内 DNS 解析）：

```yaml
spring:
  datasource:
    url: jdbc:mysql://huashui_mysql:3306/huashui_campus_life?...
  data.redis.host: huashui_redis
  rabbitmq.host: huashui_rabbitmq
  cloud.nacos.discovery.server-addr: huashui_nacos:8848
```

```bash
docker compose ps                          # 查看所有容器状态
```

```bash
# 停止所有容器（保留数据）

# 停止并删除所有数据（完全重置，谨慎操作）

       ~/water_compus/rabbitmq/data ~/water_compus/nacos/data \
       ~/water_compus/minio/data
```

```bash

  --all-databases --single-transaction --quick \
  > backup_$(date +%Y%m%d_%H%M%S).sql
```

## 10. 常见问题排查

### Q1: Nacos 启动后报 "No DataSource set"

**原因**：MySQL 未就绪时 Nacos 已启动。

**解决**（VM 和 WSL 通用）：
```bash
docker compose restart huashui_nacos
```

### Q2: XXL-JOB 页面可以打开但报数据库错误

**原因**：`xxl_job` 库未创建或表未初始化。

**解决**：
```bash
docker exec -it huashui_mysql mysql -uroot -proot -e "USE xxl_job; SHOW TABLES;"
```
如果表不存在，手动执行 `mysql/init/01-create-databases.sql`：
```bash
docker exec -i huashui_mysql mysql -uroot -proot < mysql/init/01-create-databases.sql
```

### Q3: 端口冲突（3306/6379 等已被占用）

**原因**：本机已有 MySQL/Redis 等进程。

**解决**：修改 `docker-compose.yml` 中对应服务的 `ports` 左侧端口号，如 `"3307:3306"`，并同步修改 Spring Boot 配置。

### Q4: WSL — 容器启动后数据库写入极慢 / 报文件锁错误

**原因**：项目放在 `/mnt/c/`（Windows 文件系统），Docker 跨文件系统挂载性能差。

**解决**：
```bash
# 将项目复制到 WSL 内部文件系统
cp -r /mnt/e/Workspace/demos/water_compus ~/water_compus
cd ~/water_compus
docker compose down
docker compose up -d
```

### Q5: WSL — `docker: command not found`

**原因**：Docker Desktop 未运行，或 WSL Integration 未开启。

**解决**：
1. 确保 Docker Desktop 正在运行（任务栏有鲸鱼图标）
2. Docker Desktop → Settings → Resources → WSL Integration → 勾选对应发行版 → Apply & Restart

### Q6: WSL — 容器数据在 Windows 重启后丢失

**原因**：WSL 默认不会随 Windows 关机而持久化数据目录。

**检查**：WSL 内部文件系统（`~/water_compus/`）的数据是持久的，不会丢失。如果使用了 `/mnt/c/` 路径且手动删除了目录，数据才会丢失。

### Q7: VM — 云服务器上无法从外网访问 Nacos / XXL-JOB

**解决**：
```bash
# 1. 检查防火墙
sudo firewall-cmd --list-ports

# 2. 开放端口
sudo firewall-cmd --add-port=8848/tcp --permanent
sudo firewall-cmd --add-port=8081/tcp --permanent
sudo firewall-cmd --reload

# 3. 检查云服务商安全组规则（阿里云/腾讯云/华为云控制台操作）
#    确保入方向放行对应端口
```

### Q8: VM — `Got permission denied while trying to connect to the Docker daemon socket`

**原因**：当前用户不在 docker 组中。

**解决**：
```bash
sudo usermod -aG docker $USER
# 退出重新登录，或执行: newgrp docker
```
